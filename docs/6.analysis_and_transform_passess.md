# VOLT Analysis and Transform Passes

## 1. Clang Flags

- `+vortex`: Enables Vortex code generation during compilation.
- `zicond`: Enables the LLVM lowering pass that converts the ternary operator into the Vortex `VX_CMOV` instruction.

## 2. LLVM Optimization Passes

### `vortex-branch-divergence`

Sets the Vortex branch divergence mode.

- `0`: Disable branch divergence handling.
- `1`: Enable branch divergence handling, structurizing only non-regional divergent branches first.
- `2`: Enable branch divergence handling, structurizing all divergent branches first.

### `vortex-kernel-scheduler`

Sets the Vortex kernel scheduler mode.

- `0`: Disable the scheduler.
- `1`: Enable the scheduler.

### `vortex-branch-opt`

Sets the Vortex branch divergence optimization level. This can also be controlled via the `VORTEX_DIVERGENCE_OPT_LEVEL` environment variable.

- `1`: Enable basic transformations.
- `2`: Enable uniformity analysis. This includes always-uniform value analysis based on the hardware structure in the divergence tracker.
- `3`: Enable uniform annotation. This analyzes metadata-based annotations from the front-end compiler and user-provided annotations.
- `4`: Enable function argument analysis. This extends uniformity analysis to function arguments, which are typically treated conservatively as divergent.
- `5`: Enable `zicond`. This applies optimizations such as lowering the ternary operator to the Vortex `VX_CMOV` instruction.
- `8`: Default.
- `9`: Enable the reconstruction pass.

## 3. Front-End Scheduling Code Generation Passes

### `VORTEX_SCHEDULE_FLAG`

Vortex kernel scheduling flags:

- `0`: **Thread mapping.** Map each OpenCL work-group / CUDA block to one Vortex thread.
- `1`: **Core mapping.** Map each OpenCL work-group / CUDA block to one Vortex core.
- `2`: **1-to-1 mapping.** Map each OpenCL work-group / CUDA block to one Vortex core (Fast). This does not cover cases where the work-group size is larger than the number of Vortex threads in a single Vortex core (`#warps × #threads` per core).
