## 2. Changing the Vortex Architecture and Configuration

One of Vortex’s key strengths is its highly reconfigurable architecture. Users can tune GPU core pipeline parameters (e.g., number of warps, threads, and issue width), the hierarchical organization (e.g., number of cores, sockets, and clusters), and the memory hierarchy and features (e.g., enabling local memory and L1/L2/L3 caches, as well as configuring cache capacities).

To modify the Vortex architectural configuration, use the `ci/blackbox.sh` script from the Vortex build directory. For the full set of supported configuration options, refer to the configuration usage script ([`ci/regression.sh.in`](https://github.com/vortexgpgpu/vortex/blob/master/ci/regression.sh.in)) and the generated hardware configuration header in the build tree (`$VORTEX_HOME/build/hw/VX_config.h`).

```bash
./ci/blackbox.sh --cores=2 --warps=32 --threads=32 --l2cache
./ci/blackbox.sh --cores=2 --warps=32 --threads=32 --l2cache --perf=1  # general performance summary
./ci/blackbox.sh --cores=2 --warps=32 --threads=32 --l2cache --perf=2  # memory performance summary
./ci/blackbox.sh --cores=2 --warps=32 --threads=32 --l2cache --debug=1 # debug flag from 0 to 5
```
Vortex supports both 32-bit and 64-bit targets. Select the target by setting `xlen=32` or `xlen=64` in the Vortex configuration. To build a 32-bit or 64-bit toolchain, set `--xlen=32` or `--xlen=64` during configuration:

```bash
# 64-bit toolchain build
../configure --xlen=64 --tooldir=$TOOL_DIR --prefix=$VORTEX_PREFIX

# 32-bit toolchain build
../configure --xlen=32 --tooldir=$TOOL_DIR --prefix=$VORTEX_PREFIX
```

Also set the environment variable used by the VOLT toolchain so the compiler and related utilities select the correct XLEN:

```bash
export XLEN=64
# or
export XLEN=32
```
